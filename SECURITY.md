# Security Policy

## Поддерживаемые версии

Проект пока в раннем развитии. Поддерживается только последняя версия из `main`.

| Версия | Поддержка |
|--------|-----------|
| 0.1.x  | ✅        |
| < 0.1  | ❌        |

## Как сообщить об уязвимости

Если нашёл что-то, что потенциально угрожает приватности или безопасности пользователей — **не публикуй в Issues**. Напиши напрямую мейнтейнеру или открой [Security Advisory](../../security/advisories/new) в GitHub.

Типичные примеры того, что интересует:
- Утечка данных куда-то наружу (сетевая активность, логи)
- Возможность выполнить произвольный код через имя файла
- Обход прав доступа macOS
- Проблемы с правами на конфиг/скрипты

Мы постараемся ответить в течение 7 дней и обсудить координированное раскрытие.

## Что НЕ считается уязвимостью

- Whisper иногда выдумывает фразы — это особенность модели, не баг
- Записи доступны всем, у кого есть доступ к твоему Mac — это by design
- Модели скачиваются с HuggingFace при установке — это единственная сетевая операция, по замыслу

## Принципы безопасности проекта

- **Офлайн-работа** после первичной установки — никаких API-вызовов в runtime
- **Локальное хранение** — всё в `~/Recordings`, никакой синхронизации без явного согласия
- **Минимум прав** — скрипты работают от имени пользователя, не требуют sudo (кроме brew install)
- **Открытый код** — любой может проверить, что никуда ничего не уходит
